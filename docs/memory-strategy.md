# Contextual Memory Strategy

When a conversation exceeds 20 messages, the chatbot no longer sends the full raw history to the LLM. Instead, it generates a concise running summary of every message beyond the most recent 10 using a separate LLM call (Groq's `llama-3.1-8b-instant`). This summary is stored directly on the Conversation document as a `contextSummary` field. On each subsequent request, the summary is prepended as a system-style message ("Summary of earlier conversation: …") before the last 10 raw messages, keeping the prompt window compact while preserving early-context facts, decisions, and preferences. The summary is regenerated lazily — after each assistant response is saved, a fire-and-forget check counts the messages and triggers a re-summarization if the threshold is crossed, never blocking the streaming reply. This approach keeps token usage bounded to ~15 messages worth of context regardless of conversation length, and avoids the complexity of embedding-based RAG for most use cases.
